Reference Model
Digital Twin

The Digital Twin is decoupled from n8n and Power BI per instructor feedback. It serves as a visualization and simulation reference layer only.

What it contains

  • 3D model of Bay & Equipment
  • Static information (tag_name)
  • Bay layout, capacity, connections
  • Station (Bay) static information
  • Reference to Odoo EPP Master Data

Purpose

  • Visualize & simulate operations
  • Decision support (future use)
  • Not connected to n8n or Power BI
  • Designed using DTDL standard
Data Flow
End-to-End Pipeline

PLC → Kepware → Azure IoT Hub

Machine signals travel OPC-UA → MQTT → cloud ingestion. The real-time pipeline starts here.

Stream Analytics flattens arrays

tas-stream-job transforms nested array payloads into flat field_id / field_value rows.

Bronze → ETL → Silver (every 5 seconds)

Raw data lands in Blob Storage. ETL Timer Function transforms to 6-row BayStatus_Realtime format and writes to Azure SQL every 5 seconds.

Azure Function → MongoDB → n8n → LINE

Events trigger serverless functions. Data flows into MongoDB for n8n to orchestrate AI agent responses via LINE messaging.

Power BI DirectQuery on Silver Layer

Dashboard queries Azure SQL directly — no scheduled refresh. Near real-time analytics for management.

Data Types
Signal Classification

Real-Time (Telemetry)

Live machine readings from PLC via IoT Hub and Stream Analytics.

Transactional (ERP / Ops)

Order and operational records from Odoo ERP via Power Apps connector.

Alert / Notification

Event-driven messages triggered by n8n and delivered via LINE API.

User Input / Command

Manager queries and commands received through LINE back into the n8n agent.
